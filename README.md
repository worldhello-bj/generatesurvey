# 大量调查数据人工智能生成器（最终版）

## 1. 项目概述

构建一个基于 AI 的合成调查数据生成系统。用户上传调查问卷，系统利用 AI 模型解析问卷结构，并根据预设的社会人口统计学特征生成对应人群的提示词，通过并行调用多个 AI 模型 API 模拟不同人群的回答，最终将生成的答案清洗、聚合为结构化数据表格（CSV/Excel），供用户下载。

系统设计核心原则：
- 用户数据不长期存储；
- 生成结果仅在内存中临时处理；
- 用户下载后数据立即删除，确保隐私安全；
- 提供管理员界面，用于监控系统 token 花费和用户流量等运营数据。

## 2. 系统架构

采用前后端分离的无状态服务架构：
- 用户数据（问卷、生成结果）仅在处理流程中驻留于内存或临时存储，任务结束后自动清理；
- 运营数据（token 花费、API 调用次数等）持久化存储，供管理员界面查询。

核心模块：
- 前端界面：用户端（上传、参数配置、进度、下载）+ 管理员端（登录、仪表盘、监控）
- 问卷解析服务：AI 解析问卷结构，数据不落盘
- 人群配置与提示词引擎：基于内置人群库生成提示词
- AI 生成与并行调度服务：异步并发调用模型 API，并记录 token/花费
- 结果清洗与表格生成服务：生成 CSV/Excel 并提供一次性下载 URL
- 运营数据服务：接收并持久化运营记录
- 管理员后台：可视化展示 token 花费、用户流量等指标

系统支持水平扩展，服务无状态；运营数据库独立部署。

## 3. 核心功能模块详述

### 3.1 问卷解析模块（AI 驱动）

处理流程：
1. 文本提取（OCR 可选）
2. 调用 AI 结构化解析并返回 JSON
3. 格式校验与修正

异步上报运营数据（任务类型 `parse_questionnaire`）：
- 时间戳
- 用户标识（会话 ID 或 IP，仅统计用途）
- 模型名称
- token 使用量（输入 + 输出）
- 估算花费
- 成功状态

数据策略：解析后的问卷 JSON 仅保存在内存，与任务 ID 关联，不写入磁盘/数据库。

### 3.2 人群配置与提示词引擎

- 在内存中按样本量与比例生成人群标签列表
- 组合提示词（模板 + 人群标签 + 问卷 JSON）
- 推入带 TTL 的 Redis 队列
- 本模块不直接调用 AI，无花费上报

### 3.3 AI 生成与并行调度服务

- 支持多供应商、多密钥轮询与速率限制
- 从队列拉取任务并异步并发调用 AI 模型

每次调用后异步上报运营数据（任务类型 `generate_response`）：
- 时间戳
- 用户标识（与解析任务同一会话 ID）
- 模型名称
- token 使用量（输入提示词 + 输出答案）
- 估算花费
- 成功状态
- 可选人群标签（用于成本分布分析，注意隐私）

数据策略： 原始 AI 回复暂存 Redis（默认 TTL 30 分钟，可配置），传递清洗服务后即删除。TTL 应根据任务平均耗时、队列峰值与系统 SLA 调整。

### 3.4 结果清洗与表格生成模块

- 读取 Redis 原始回复并清洗
- 生成 DataFrame 后导出临时文件（tmpfs）
- 提供一次性下载链接，过期自动清理
- 可选记录文件大小、样本量等运营指标

### 3.5 运营数据服务

- 通过异步 API（HTTP POST / 消息队列）接收运营记录
- 写入 PostgreSQL 持久化存储

建议表结构字段：
- `id`（主键）
- `task_type`
- `user_id`
- `model`
- `timestamp`
- `prompt_tokens`
- `completion_tokens`
- `total_tokens`
- `cost`
- `success`
- `metadata`（JSON，可选）

数据保留： 默认保留 90 天并定期归档/清理，并通过配置项调整以满足业务与合规要求（如本地数据保护法规）。

### 3.6 管理员后台

功能：
- 管理员认证（JWT 或 Session）
- 实时概览：今日花费、调用次数、活跃用户数、平均响应时间
- 花费趋势：7/30 天折线图，可按模型分组
- 流量分析：按时间请求量、任务类型占比、模型使用排行、可选地域分布
- 详细记录：支持时间/用户/模型/任务类型筛选与分页查询
- 成本控制：月度预算阈值与告警（可选）

技术实现：
- 后端提供 RESTful API（聚合、筛选、分页）
- 前端图表库（ECharts/Chart.js）
- 热点查询可通过 Redis 缓存

### 3.7 前端界面（用户端 + 管理员端）

- 用户端保持原流程
- 管理员端独立路由（如 `/admin`），必须登录访问
- 可使用 Ant Design Pro 快速搭建

## 4. 技术栈补充

| 层次 | 技术选择 |
| --- | --- |
| 运营数据库 | PostgreSQL（持久化） |
| 缓存 | Redis（临时队列 + 运营缓存） |
| 后端框架 | Python FastAPI（异步） |
| 管理员前端 | React + Ant Design Pro + ECharts |
| 身份验证 | JWT 或 Session |
| 消息队列（可选） | Redis Stream 或 RabbitMQ |

## 5. 数据生命周期与隐私保护机制

- 运营数据仅存储系统级指标，不包含问卷内容、生成答案或个人身份信息
- 用户标识采用临时会话 ID 或哈希 IP，避免反向追溯
- 用户数据遵循零存储原则：处理完成即销毁
- 日志脱敏，避免意外记录用户内容

## 6. 关键挑战与解决方案

1. **上报性能影响**：使用异步队列/流批量写库，避免阻塞主流程  
2. **花费估算准确性**：维护模型价格配置中心并定期更新  
3. **管理员安全性**：强认证 + `/admin/*` 权限中间件，建议默认启用双因素认证（2FA）  
4. **数据增长**：按时间分区、建立索引、定期归档冷存储

## 7. 开发阶段规划

### 阶段一：MVP
- 用户端核心流程（解析 + 少量同步生成）
- token 花费先记录日志
- 无管理员界面

### 阶段二：增强 + 运营基础
- 异步并发 + Redis 临时队列
- PostgreSQL 持久化运营数据（异步上报）
- 基础管理员 API + 简单前端展示

### 阶段三：管理员完善
- 完整仪表盘（图表、筛选、导出）
- 管理员认证
- 缓存与性能优化
- 云部署与安全组配置

## 8. 总结

本项目通过 AI 驱动问卷解析与答案生成，实现从非结构化问卷到结构化数据的自动化流程，并在全链路中坚持用户数据零存储。通过新增管理员界面与运营数据服务，系统具备完整可观测性，可支持成本控制、流量监控和持续优化。

## 9. 快速启动脚本

仓库根目录提供 `start.sh`，用于一键通过 Docker Compose 启动全部服务。

```bash
chmod +x ./start.sh
./start.sh
```

后台启动：

```bash
./start.sh -d
```

首次运行若不存在 `.env`，脚本会自动基于 `.env.example` 生成 `.env`，请先补充有效配置后再重新执行启动命令。
